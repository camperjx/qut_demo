Assignment overview

This second assignment will have you create a basic Web page that allows a user to interact with a pair of related datasets to perform basic filtering and analysis with them. It is intended to test your understanding of all modules in this unit, but with a focus on the concepts and techniques introduced in the second half.

You will be assessed on both your ability to complete the tasks given below, as well as a more general understanding of the concepts of programming and JavaScript that you have been taught.

This assignment supports unit learning outcomes 1, 2, 3 and 4.

Assignment details

For this assignment, we are looking for the following two items in your submission:

  • The code and data files used in your solution, in a zip archive. Code files should be using a .js extension, and Web pages should be using a .htm or .html extension.
  • A report describing your solution, in a DOC, DOCX or PDF file.

If you have downloaded any modules for use with your solution, you do not need to include the node_modules folder or the package-lock.json file in your archive. These can be installed when we test our solution. If you have any code that does not work but is part of an attempt to complete one of the tasks, you can still include it. In this case, you should provide comments indicating the problematic code, and/or include it in a separate file.

Your report should inform us of all the tasks you implemented in your solution, as well as those that you attempted to implement or could not. There should also be details of any errors that may occur in your code that you were not able to fix before submission. The results of your Jest unit tests should also be included in your report.

Data source

The dataset we will be using for this assignment is based on the OpenFlights collection of airports, routes, airline and aircraft databases, available with an Open Database License that allows free use as long as the source is acknowledged and any public derivations of the work are put under the same license. OpenFlights' data source for routes ceased functioning in 2014, so the route data is likely outdated but should not be an issue for this assignment.

We have narrowed down this dataset to airports and routes only for the purposes of this assignment, and only those routes that start and end in Australia. This does include routes that have a codeshare with other airlines (though does not explicitly say which airline is codesharing with the other).

The data provided is in JSON format, in two files: A2_Flights.json and A2_Airports.json, having information on the Australian routes and airports respectively.

HTML cheat sheet

Here are some HTML tags and elements that you may see or use on the Web page for this assignment.

  • <script></script> â€“ JavaScript to be used on the page. To use an external script, include an "src" attribute with the path (if it is not in the same folder) and filename to that script. (Leave the inner HTML empty in this case.)
  • <body></body> â€“ The tags that contain the elements of the page that should be displayed in the browser.

Task description

Consult the following task description table for a more detailed view of what this assignment entails.

Step 1: Loading data sources

As with Assignment 1: Programming exercises, we will want you to load the data we have provided into your JavaScript program for use later. However, since it is in two separate data files, you will need to combine this data together once you have parsed it. This will make it easier to work with later when we do our analysis.

Your merged dataset should have the following characteristics:

  • Include the "codeshare" and "aircraft" properties as is.
  • Include the data for both the source and destination airports, under the "source_airport" and "destination_airport" properties. This can be done in two ways:
  • Create a new object for each property that is a copy of the airport data for the corresponding airport (including all properties that are in the airports JSON); or
  • Keep the airports as a separate array and make each property a reference to the corresponding airport in that array.
  • Include the data of the airlines in one of two ways:
  • Keep them as individual properties in the flight object: "airline", "airline_name" and "airline_country"; or
  • Create a single "airline" property that contains an airline object that has the code, name and origin country.

Step 2: Mapping function for data

You should also have a function that can be used to map the data once it is combined, or in other words, allow us to perform a modification across all elements of the whole dataset. Ideally, this would allow us to specify a certain operation to perform on each member, in the form of a function parameter.

This mapping function should also provide a timestamp for the operation you performed on the data. If the data has a timestamp from a previous mapping operation, it should be updated. However, this should not alter the original data itself.

If you are keeping a separate copy of the airport data on its own, you are only required to provide a mapping function for the flights, though you are welcome to include a mapping function for airports as well.

Step 3: Data analysis

After creating your mapping function, you will want to provide the functions to pass to it that perform the actual analysis on the combined dataset.

As in the previous assignment, there are several tasks that you can perform:

  • Display information on all flights in the dataset.
  • Display all flights that have certain criteria. For example, all flights that originate from Brisbane, or which flights use a Boeing 737.
  • Determine average, minimum and maximum values based on the flights. For example, which pairs of airports have the most flights, or which flights have the most types of aircraft flying on them?
  • Calculate new characteristics based on the given data. For example, the direct distance between airports on a route based on their latitude and longitude.

Since a mapping function should return a new copy of the array it is processing, not all your functions will need to be used as a callback to it. Feel free to make non-mapping functions to complete some of these tasks.

You should attempt to create your functions as closely as possible to a Functional style:

  • Your functions should include all variables/values they use as arguments, and produce the same result with the same input (i.e. pure functions); and
  • Your functions should avoid changing any values outside of the function body, avoiding side effects that can affect other functions (i.e. immutability).

Step 4: Interactive web page

Our previous assignment did not include any interactivityâ€”we specified all the analysis by directly calling the functions we made for the dataset. This time, we want a user to be able to explore the dataset themselves, hence you will need to provide a Web page that interacts with the data in various ways.

The user should be able to access some of the filtering and analysis functions that you created in the previous tasks and see the results presented on the web page. If possible, any processing you do in the previous task should be accessible by a user to be shown on the web page.

Since we have not covered creating a Web page using HTML to a great extent during this unit, we have provided you with a basic HTML file with most of the elements already present to interact with and display data from the datasets we have provided. There is also a quick reference to basic HTML elements available later.

If you need to add any elementsâ€”either by editing the HTML or by adding them through JavaScriptâ€”you may do so. Any values related to the dataset should be added via JavaScript rather than hardcoded into the Web page, if possible.

Note: Refer back to the HTML cheat sheet in the Data set accordion.

We are after functionality rather than presentation, so you do not need to worry about your solution's appearance in the browser; only ensure any content you show is understandable. If you wish, however, you are welcome to style your Web page to improve its appearance.

Step 5: Unit tests

As before, we wish you to provide unit tests to prove that your functions work with a variety of possible inputs.

As much as possible, all your functions should have at least one unit test to ensure they work as expected. Those functions that accept inputs should have a variety of different inputs testedâ€”standard values, extreme values and erroneous valuesâ€”to demonstrate how they are handled by your program. Your functions should be able to gracefully handle erroneous input without stopping, which is especially important now that we are letting users input data.

Step 6: Functional JavaScript

While we do not expect you to write your whole code in a functional style (and this is not possible since we are modifying a Web page), since we have demonstrated functional principles during the unit, we would also like to see you apply them to some of the code that you submit for this assignment.

Remember that the two principles of functional programming are:

  1. Pure functions that always return the same output with the same input, and do not have side effects on the rest of the program state; and
  2. Immutability with objects and variables that are not changed after they are created, and any needed changes are returned as new values.

For example, we have suggested that your mapping function should not affect the original data set after you have applied the passed function to it.
